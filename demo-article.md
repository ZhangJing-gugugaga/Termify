# Termify — 万物皆可终端

## 【产品名称】

Termify（万物终端化转换器）

## 【一句话介绍】

给所有想把 GIF、图片变成终端动画和单片机屏幕画面的创作者，一键上传、选风格、导出可播放的代码文件。

## 【原型展示】

【封面】

Termify 是一个开源的 Web 小工具。用户上传一张图片或一段 GIF，选择喜欢的字符渲染风格，就能下载一个可以直接在终端播放的 Python 脚本、一个打开即播的 HTML 页面，甚至是一份能烧进 Arduino / ESP32 单片机 OLED 屏幕的 C 代码。

它不解决严肃的生产力问题——它解决的是"我想把我的猫表情包显示在终端里"这种快乐的问题。

【上传】

## 【核心功能】

这次原型展示的是"任意图片/GIF → 终端动画"的完整转换流程。

产品提供 5 种字符渲染风格：经典 ASCII 灰度（@#%*+=-:.）、Unicode 色块（█▀▄+TrueColor 真彩色）、Braille 点阵（⠁⠂⠄⡀ 高分辨率）、几何图形（■□◆◇ 现代感）、极简二值（纯黑白）。用户可以在页面上实时预览每种风格的效果，选定后下载对应格式的动画文件。

特别地，嵌入式设备输出支持 4 种常见单片机屏幕分辨率预设（SSD1306 OLED 128×64、ST7735 TFT 160×128 等），生成 PROGMEM bitmap 数组和 Arduino 绘图代码，拿来改改引脚定义就能烧录运行。

## 【用户使用流程】

1. 用户打开 Termify 首页，看到终端风格的深色界面和一行 CLI 演示命令，3 秒理解产品是做什么的。
2. 拖拽或点击上传一张 GIF / PNG / JPG 图片（最大 20MB）。
3. 在 5 种渲染风格卡片中选择一种，点击后实时预览该风格下的静态效果。
4. 在预览区查看动画播放效果，可以暂停、拖动进度条、查看帧数。
5. 选择输出格式：Python 脚本（.py）、HTML 页面（.html）、或嵌入式设备（Arduino .ino / C 头文件 .h）。如果选择嵌入式，展开 MCU 子面板选择目标屏幕分辨率。
6. 点击下载，拿到可以直接运行的代码文件。

## 【AI 的作用】

在当前 MVP 阶段，"AI"主要体现在图像处理的智能适配上：系统自动分析上传图像的像素分布，通过 Floyd-Steinberg 抖动算法将连续色调映射到目标字符集，并根据目标终端尺寸智能缩放——确保在任何分辨率下都保留最多的视觉细节。

下一步计划引入真正的 AI 能力：用图像理解模型自动识别图片中的主体区域，在字符映射时给予更高权重，让"脸"比"背景"保留更多细节。

## 【下一步迭代计划】

- 接入视频输入（MP4），支持更长动画的转换
- 增加用户历史记录功能，方便重新下载或分享
- 优化嵌入式输出，支持更多开发板（Raspberry Pi Pico、ESP32-S3）和彩色 TFT 屏幕
- 增加在线分享页面，生成链接即可在浏览器中预览终端动画
- 引入 AI 主体识别，提升字符映射的视觉效果
- 开放 Issue 驱动的用户自定义字符集，社区投票决定新增哪些风格
