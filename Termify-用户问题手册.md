# Termify 用户问题手册

> 帮你从「看不懂」到「玩得转」——像朋友一样排坑
> 适用版本：Termify v1.0（GIF/PNG/JPG → 终端动画）
> 编写：许清楚（产品经理）｜日期：2026-07-17

本手册按你的水平分三层，直接翻到对应的那层就行：

- **小白（0 技术外行）**：连「终端」「Python」是啥都不知道，只想把喜欢的图变成动画发出去。
- **初级用户（会一点）**：装过 Python、敲过几行命令，能跑通但效果/分享上有疑问。
- **高级用户（开发者/集成者）**：要批量调用、扩展风格、部署上线、搞嵌入式。

每条都是「问题 / 现象 / 解决 / 备注」四段式，能直接抄的命令都给你标好了。

---

## 一、小白篇（0 技术外行）

> 你大概率**不需要**装 Git、装 Python、敲命令。Termify 有两种省心用法：① 等官方「在线 Demo」（打开网址即用）；② 下载别人打好的「一键包」（双击即用）。下面的坑，主要是为了帮你判断「为什么我这里不行」以及「怎么发给朋友」。

### 1. git 是什么？我需要装吗？
- **问题**：网上教程一上来就让你 `git clone`，git 到底是啥？
- **现象**：看不懂，不知道要不要装。
- **解决**：git 是个「下载别人代码的工具」。但普通用户**不需要**它。只有开发者才用 git 去拿源代码。你等「在线 Demo」或「exe 一键包」即可。
- **备注**：别被教程里的 git 命令吓退，那不是给你准备的。

### 2. Python 环境怎么装？
- **问题**：教程说「先装 Python」，去哪下、怎么装？
- **现象**：装完在终端敲 `python` 没反应。
- **解决**：去 [python.org](https://www.python.org) 下载 Python 3.10+ 安装包；安装时**务必勾选最底下的 "Add Python to PATH"**。装完打开「终端/命令行」敲 `python --version`，能显示版本号就成功了。
- **备注**：如果用官方「一键包」，连 Python 都不用装。

### 3. 终端（命令行）是什么，怎么打开？
- **问题**：总说「在终端里运行」，终端在哪？
- **现象**：Windows 找不到，Mac 不知道叫啥。
- **解决**：
  - Windows：按 `Win` 键输 `cmd` 或 `powershell`，推荐装微软商店免费的「Windows Terminal」；
  - Mac：启动台搜「终端」；
  - Linux：按 `Ctrl+Alt+T`。
  它就是那个黑底白字的窗口，你在里面敲命令让电脑干活。
- **备注**：想看彩色动画，Windows 用户**务必**用 Windows Terminal，老版 cmd 会乱码（见第 6 条）。

### 4. .html 是什么，怎么打开看动画？
- **问题**：下载的 .html 双击打不开 / 打开了是代码？
- **现象**：双击后用记事本打开了（全是英文代码），或浏览器打开了但没动。
- **解决**：.html 是网页文件。右键它 →「打开方式」→ 选 **Chrome / Edge / 火狐浏览器**。正确打开后动画会自动播放。
- **备注**：.html 最大好处——朋友**不用装任何东西**，发微信/邮件给他，浏览器打开就能看。

### 5. 我下载的 .py 文件双击打不开 / 一闪而过，怎么回事？
- **问题**：双击 .py，黑窗口闪一下就没了。
- **现象**：以为坏了，其实是「跑完就关」。
- **解决**：.py 是 Python 脚本，**不能直接双击**，要在终端里跑：
  ```bash
  cd 文件所在的文件夹
  python 你下载的文件名.py
  ```
  比如 `python termify.py`。先 `cd` 到下载目录，再敲命令。
- **备注**：你朋友要看到这个动画，也得装 Python 并在终端跑。嫌麻烦就选 **.html** 格式发给朋友。

### 6. 为什么朋友的彩色动画我这里是一堆乱码/方块？
- **问题**：朋友发来的彩色 .py，在我电脑上是方块、乱码、颜色全丢。
- **现象**：Windows 老版 cmd 尤其明显。
- **解决**：这是「终端不支持 TrueColor（24 位真彩）」导致的。两个办法：① 改用 **Windows Terminal / PowerShell** 打开；② 让朋友重新生成**非彩色风格**（如「经典 ASCII 灰度」「极简二值」）的 .py，或改发 **.html** 版本（浏览器不挑终端）。
- **备注**：Termify 后续会「自动降级」——检测到旧终端就提示用 HTML 或换非彩色风格。

### 7. 上传文件时提示「上传失败 / 文件太大」怎么办？
- **问题**：选了文件点上传，报错。
- **现象**：提示「文件过大」「格式不支持」或「网络错误」。
- **解决**：
  - **格式**：只支持 GIF / PNG / JPG，其他（MP4、zip、pdf）不行；
  - **大小**：超过 **20MB** 会被拒，先压小或裁短；
  - **网络**：在线版加载慢就多试几次，或换本地版。
- **备注**：更友好的错误提示是 v2 优化点。

### 8. 页面上 5 张风格卡片看不懂，我该选哪个？
- **问题**：ascii / blocks / braille / geometric / binary 都是啥？
- **现象**：全是英文术语，不敢点。
- **解决**：大白话版——
  - **经典 ASCII 灰度**：最复古的「字符画」，黑白灰，任何终端都能看（最稳）；
  - **Unicode 色块**：彩色方块拼的，最好看，但要 TrueColor 终端；
  - **Braille 点阵**：盲文点拼的，很细腻、很「科技感」；
  - **几何图形**：方块圆圈菱形，现代简约风；
  - **极简二值**：纯黑白方块，老报纸印刷感。
  新手先试「经典 ASCII 灰度」或「Unicode 色块」看预览。
- **备注**：每张卡片都能点开看实时预览，挑顺眼的再下载。

### 9. 什么是 charset（字符集）、TrueColor、终端尺寸（如 40×20）？
- **问题**：页面选项一堆名词，啥意思？
- **现象**：不懂参数，不敢乱点。
- **解决**：
  - **charset（字符集）**：就是上面那 5 种「风格」的学名；
  - **TrueColor（24 位真彩）**：让终端显示上千万种颜色，彩色风格靠它；老终端不支持就会乱码（见第 6 条）；
  - **终端尺寸（宽×高，如 40×20）**：动画的「字符格子」数量，数字越大越精细但越占屏、越卡。
- **备注**：小白用默认值就好，不用纠结。

### 10. 做好的动画怎么发给朋友看？
- **问题**：我做出来了，怎么分享？
- **现象**：不知道发 .py 还是 .html，怕朋友打不开。
- **解决**：
  - 想让朋友**零门槛看**：选 **.html**，发微信/邮件，朋友浏览器打开就行；
  - 想让朋友**在终端里看**（极客风）：选 **.py**，但朋友得装 Python 并在终端跑（见第 5 条）。
- **备注**：v2 会做「一键分享链接 / 社区画廊」，到时候直接发链接。

### 11. 我电脑不行，跑不起来 / 打不开网页，有没有更简单的办法？
- **问题**：按教程装了一堆还是用不了。
- **现象**：卡在某一步，想放弃。
- **解决**：等官方 **在线 Demo**（打开网址即用）或 **一键 exe / AppImage 包**（下载双击即用，无需 Python）。v1 的本地版对纯小白门槛偏高，这是已知问题，正在修。
- **备注**：关注项目 GitHub 的 Releases 页面拿打包好的版本。

### 12. 音乐怎么加？为什么我做出来的没声音？
- **问题**：别人的有背景音乐，我的没有。
- **现象**：下载的 .py 静音，或 .html 没声音。
- **解决**：
  - .py 版：把一首音乐文件**重命名为 music.mp3**，放在 .py 同目录，播放时自动响；
  - .html 版：浏览器自动播放策略限制，音频支持还在完善。
- **备注**：音乐文件要正好叫 `music.mp3`；没有就静音播放，不影响动画。

### 13. 这个东西要钱吗？安全吗？我的图片会不会被上传到别人服务器？
- **问题**：怕收费、怕泄露。
- **现象**：担心隐私。
- **解决**：
  - **免费**、MIT 开源；
  - 本地版：文件只在你电脑处理，不上传；
  - 在线版：文件会传到服务器处理，但项目不存用户数据、处理完即删（零注册）。
- **备注**：敏感图片建议用本地版或离线包。

## 二、初级用户篇（会一点）

> 你装过 Python、敲过命令，能跑通基础流程，但在「效果调优 / 批量 / 分享」上有疑问。

### 1. 做出来了怎么发给朋友？.py 和 .html 有什么区别？
- **问题**：选哪种格式分享？
- **现象**：不确定朋友能不能打开。
- **解决**：
  - **.html**：自包含网页，朋友**浏览器直接打开**，零门槛，首选分享；
  - **.py**：Python 终端脚本，朋友需装 Python 并在终端 `python xxx.py`，极客友好但门槛高。
- **备注**：要「对方一眼能看到」就发 .html；要「在终端里秀」才发 .py。

### 2. 效果好卡，播放一卡一卡的怎么办？
- **问题**：预览/播放卡顿。
- **现象**：200×60 这种大尺寸尤其卡；文件大、请求慢。
- **解决**：
  - 调小**终端尺寸**（如从 200×60 降到 80×24 或 120×36），数字越小越流畅；
  - 帧数太多可降采样；
  - 用 .html 播放（前端已做预渲染优化，比 .py 终端刷新稳）。
- **备注**：超大尺寸会触发「全帧返回卡顿」（200×60×100 帧≈6MB 单请求），v2 会做分块加载/默认降档。

### 3. 效果好丑 / 颜色怪，是不是风格选错了？
- **问题**：出来不是想要的效果。
- **现象**：彩色变灰、方块错位、像马赛克。
- **解决**：
  - 想要彩色却显示灰白 → 终端不支持 TrueColor，换 **Windows Terminal** 或改用非彩色风格；
  - 想要细腻却很糙 → 尺寸太小，调大；或风格不适合（照片用 ASCII/blocks，线条用 braille）；
  - 颜色错位 → 多半是 blocks 半色块在旧终端失效，换终端或换风格。
- **备注**：先在某风格卡片上点「预览」看真实效果再下载。

### 4. 好模糊 / 看不清，怎么调清楚？
- **问题**：细节丢了很多。
- **现象**：缩太小、风格颗粒粗。
- **解决**：
  - 调大**终端尺寸**（宽高）；
  - 照片类用 **blocks（彩色色块）** 或 **ascii** 更保真；
  - 确认原图分辨率够高，低清 GIF 放大也糊。
- **备注**：Termify 用 LANCZOS 高质量缩放（优于普通双线性），模糊通常是「源图本身小」或「尺寸选太小」。

### 5. 我想一次转很多个 GIF，能批量吗？
- **问题**：几十个 GIF 要转，不想一个一个点。
- **现象**：Web 一次只能传一个文件。
- **解决**：
  - **命令行（本地）**：`python demo.py 图片.gif --charset all` 可批量生成全部风格；
  - **API**：写脚本循环调 REST 接口（见高级用户第 1 条）；
  - Web 批量上传是 **P1 规划**，v1 暂不支持。
- **备注**：批量建议走 CLI 或 API，效率最高。

### 6. 怎么给我做的动画加背景音乐？
- **问题**：想让 .py 播放时有声音。
- **现象**：没声音。
- **解决**：把任意音乐文件**重命名为 music.mp3**，放在 .py 同目录，运行 .py 时自动播放（Windows 用 MediaPlayer、macOS 用 afplay）。
- **备注**：音乐不是必须；没有 music.mp3 就静音播放。

### 7. 命令行 demo.py 怎么用？有什么参数？
- **问题**：demo.py 是啥，怎么跑？
- **现象**：不会用 CLI。
- **解决**：在 Termify 目录下打开终端：
  ```bash
  python demo.py 你的图.gif                  # 默认转一个
  python demo.py 你的图.gif --charset all    # 生成全部 5 种风格
  python demo.py 你的图.gif --format html    # 指定输出格式
  ```
  具体参数以 `python demo.py --help` 为准。
- **备注**：CLI 最适合批量/自动化；Web 适合单张交互式预览。

### 8. 视频（MP4）能直接转吗？
- **问题**：想转视频。
- **现象**：上传 MP4 被拒。
- **解决**：**暂不支持直接传视频**（视频是 v2 规划）。workaround：先用工具把视频转成 **GIF**，再上传。例如 ffmpeg：
  ```bash
  ffmpeg -i video.mp4 -t 10 out.gif   # 控制时长≤10秒、大小≤20MB
  ```
- **备注**：视频直传是 P1 功能，敬请期待。

### 9. 终端尺寸（40×20、80×24、200×60）怎么选？数字越大越好吗？
- **问题**：尺寸参数怎么定。
- **现象**：不知道选多大。
- **解决**：
  - **不是越大越好**。尺寸=字符格子数，越大越精细但越占屏、越卡、文件越大；
  - 一般终端默认 80×24 够看；想高清用 120×36~200×60（需好终端+好机器）；
  - 手机/小窗用 40×20 这种小尺寸。
- **备注**：先小后大，预览满意再放大。

### 10. 我想把做好的动画做成终端登录欢迎语（banner），行吗？
- **问题**：想当 SSH/终端启动画面。
- **现象**：不知道怎么嵌。
- **解决**：可以。用 **ascii / binary** 等非彩色风格生成 .py，把 FRAMES 数据放进你的 shell 启动脚本（如 `.bashrc` 里调用），或用静态 ASCII 当 banner。彩色风格需终端支持 TrueColor。
- **备注**：非彩色风格兼容性最好，适合做 banner。

### 11. 输出文件太大了，发给朋友不方便，怎么压小？
- **问题**：.py/.html 好几 MB。
- **现象**：微信发不出去 / 加载慢。
- **解决**：
  - 调小**终端尺寸**（立竿见影）；
  - 选 **ascii / binary** 等非彩色风格（无颜色数据，体积小很多）；
  - 减少帧数（短 GIF）；
  - Termify 已用「颜色 delta 编码」把 blocks 体积从 ~12MB 压到 ~3MB，彩色风格也受益。
- **备注**：体积 vs 清晰度权衡，发朋友优先 .html + 小尺寸。

## 三、高级用户篇（开发者 / 集成者）

> 你要批量调用、扩展引擎、部署上线、搞嵌入式。下面是接口与扩展点速查。

### 1. 怎么用 REST API 程序化批量转换？给个例子。
- **问题**：不想手动点网页，想脚本批量转。
- **现象**：要自动化。
- **解决**：Termify 内置 4 个端点，典型批量流程：
  ```bash
  # 1) 上传
  curl -F "file=@cat.gif" http://127.0.0.1:5000/api/upload
  # 返回 {"task_id":"abc123", "frames_count":24, ...}
  # 2) 生成（选风格 + 格式）
  curl -X POST http://127.0.0.1:5000/api/generate \
       -H "Content-Type: application/json" \
       -d '{"task_id":"abc123","charset":"blocks","format":"html"}'
  # 返回 {"download_url":"/api/download/abc123_termify.html", "file_size":"3MB"}
  # 3) 下载
  curl -O http://127.0.0.1:5000/api/download/abc123_termify.html
  ```
- **备注**：循环遍历文件即可批量；`GET /api/preview/<id>?charset=ascii&frame=0` 可先取首帧验证效果。

### 2. 怎么让 AI agent / 脚本自动调用 Termify？
- **问题**：让 AI 工作流调用 Termify 生成动画。
- **现象**：需要程序化接口。
- **解决**：Termify 的 REST API 就是为「AI 可程序化调用」设计的。本地无网用 CLI `python demo.py 图片.gif --charset all`；有网用上面的 HTTP 端点。Python 示例：
  ```python
  import requests
  r = requests.post("http://127.0.0.1:5000/api/upload", files={"file": open("cat.gif", "rb")})
  tid = r.json()["task_id"]
  g = requests.post("http://127.0.0.1:5000/api/generate",
                    json={"task_id": tid, "charset": "blocks", "format": "html"})
  url = "http://127.0.0.1:5000" + g.json()["download_url"]
  open("out.html", "wb").write(requests.get(url).content)
  ```
- **备注**：「AI 可程序化调用」是核心集成价值，建议在 README 突出。

### 3. 怎么加第 6 种渲染风格？
- **问题**：内置 5 种不够，想扩展。
- **现象**：不知道扩展点在哪。
- **解决**：最小改动面在 `termify/charset.py`：
  1. 在 `CHARSETS` 注册元数据（name / chars / color / description）；
  2. 写 `_render_xxx()` 渲染函数；
  3. 在 `_RENDERERS` 登记该函数；
  输出层（`output/`）无需改。
  ⚠️ **双注册表陷阱**：元数据 `CHARSETS` 与函数 `_RENDERERS` 是两个字典，新手极易只改一个漏另一个，导致静默走错分支。v2 计划合并为单注册表。
- **备注**：建议配套写「如何加第 6 种风格」贡献教程（P2）。

### 4. 输出文件（.py/.html）体积怎么压小？
- **问题**：彩色 .py 动辄几 MB。
- **现象**：体积大、加载慢。
- **解决**：
  - 用**颜色 delta 编码**（相邻 cell 同色不重发转义码）：blocks 已从 ~12MB 降到 ~3MB；
  - 调小终端尺寸；
  - 非彩色风格天然小；
  - 减少帧数。
- **备注**：delta 编码参考 SalaryCat 思路，已在 v1 修复中实现。

### 5. 怎么把 Termify 部署成公开可访问的网站？
- **问题**：想让任何人打开网址就能用。
- **现象**：现在是本地自托管。
- **解决**：Flask + gunicorn + Docker 部署到现有 VPS；反向代理（nginx/Caddy）把子域名（如 `termify.moonzj.com`）指向服务，强制 HTTPS。详见 PRD「附：上线部署」。
- **备注**：内存任务存储不适合多 worker，先用单进程或加简单持久化（见技术债）。

### 6. 怎么用 Docker 部署？
- **问题**：容器化部署。
- **现象**：要可复现环境。
- **解决**：Flask + gunicorn 打 Docker 镜像，单容器跑（注意单进程避免内存存储冲突），前置 nginx/Caddy 反代 + HTTPS。
- **备注**：Dockerfile 与部署文档是 v2 交付物之一（P0 在线 Demo 配套）。

### 7. 嵌入式 Arduino / ESP32 输出做到哪一步了？
- **问题**：想输出到单片机屏幕。
- **现象**：代码里没有 embedded 输出。
- **解决**：**v1 尚未实现**，路线在 v2（M3 差异化纵深）。当前架构缺口：`FrameSequence` 是 ANSI 字符串，**缺像素级出口**。正确形态 = PC 端 `output/embedded.py` 离线生成 C 头文件/数组 + Arduino `.ino` 播放草图。需先给引擎补像素出口。
- **备注**：单色 OLED 128×64（100 帧≈100KB）轻松；RGB TFT 240×240 RGB565（≈11.5MB）超 Flash，需降采样 + 外挂 SPIFFS。

### 8. 引擎 FrameSequence 是字符串类型、缺像素级出口，我怎么扩展？
- **问题**：拿不到裸像素数据做嵌入式。
- **现象**：`engine.convert()` 算出的 `scaled`（PIL 图）被丢弃，只有 `lines_per_frame: list[list[str]]`（ANSI 字符串）。
- **解决**（推荐方案①）：给引擎加 `convert_pixels()` 或暴露 `scaled_frames`（缩放后的 PIL 图序列），由 `output/embedded.py` 读取并量化成目标屏像素数组。**不推荐**反向解析 ANSI 字符串。
- **备注**：这是嵌入式路线的前置 blocker，优先级高（M3 第一步）。

### 9. 20MB 上限、双定义、双注册表等技术债怎么修？
- **问题**：代码有重复/不一致。
- **现象**：`app.py:18` 与 `frames.py:9` 各定义 20MB（双定义）；`CHARSETS`/`_RENDERERS` 双注册表。
- **解决**：
  - 抽尺寸/上限常量到 `config.py`（单一定义）；
  - 合并 `CHARSETS`+`_RENDERERS` 为单注册表（数据+函数合一）；
  - 加单测防止回归。
- **备注**：技术债清理是 P2 / M3 工作，不影响 v1 功能。

### 10. 内存任务存储无淘汰、重启即丢，怎么持久化？
- **问题**：TASKS 全局字典，长运行吃内存、重启丢数据。
- **现象**：多 worker 冲突、内存泄漏风险。
- **解决**：单进程部署（`gunicorn --workers 1`）临时规避；长期改为简单持久化（文件 / SQLite / Redis），配 TTL 淘汰。API 限流 + 持久化是 P2。
- **备注**：在线 Demo 上线前必须处理（否则多 worker 拿不到彼此的任务）。

### 11. 预览 / 下载返回全部帧卡顿，怎么优化？
- **问题**：200×60 blocks 100 帧一次性返回 ≈6MB，前端整段渲染卡。
- **现象**：preview 无 frame 时全帧返回。
- **解决**：
  - 预览默认只取首帧 / 少量帧；
  - 前端预渲染所有帧后播放（已做，用 innerHTML 缓存）；
  - v2：分块加载、默认降一档尺寸、输出体积优化。
- **备注**：200×60 默认尺寸对 Web 偏激进，前端应引导用更小尺寸。
