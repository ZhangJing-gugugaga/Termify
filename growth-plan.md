# Termify 产品增长方案

## 【产品名称】

Termify — 万物皆可终端

## 【一句话介绍】

给开发者和硬件爱好者，把任何图片/GIF 一键转成终端动画或单片机屏幕代码的趣味转换工具。

## 【目标用户】

第一批用户画像非常清晰：程序员、嵌入式玩家、终端美化爱好者。他们活跃在 GitHub、V2EX、掘金、B站技术区、Reddit r/terminal 和 r/arduino 这些社区。这群人有几个共同特征——喜欢折腾终端主题、会在个人主页挂 ASCII art、买了一块 ESP32 但苦于没有好玩的内容往 OLED 小屏幕上放。

Termify 解决的正是这个"有屏幕没内容"的小痛点。它不是刚需，但它是那种让人看到就想试一下的玩具——而玩具的传播力往往比工具更强。

## 【增长策略】

分三个阶段：

**冷启动：做自己的第一个案例。** 用 Termify 把"月薪喵"、Doge、Hacker News logo 这些程序员圈子里的 meme 转成终端动画，录制成 15 秒 GIF，发到 V2EX、Twitter、Reddit。标题就写"I built a tool that turns any GIF into terminal art"。这类内容在技术社区的传播效率极高，因为视觉冲击力强、一看就懂、别人会想"我也要把我的猫放进去试试"。

**社区渗透：开源 + Issue 驱动。** 项目完全开源，鼓励用户提 Issue 请求新的字符集或输出格式。每一个 Issue 都是一次互动，每一次 PR 合并都是一次社区曝光。同时在 Arduino 中文论坛、ESP32 相关社群发布"如何用 Termify 给你的 OLED 屏幕做动画"的教程帖，直接触达嵌入式用户。

**内容长尾：SEO + 教程。** 写几篇博客覆盖"terminal animation generator""Arduino OLED animation""GIF to terminal"这类长尾关键词。这类工具型搜索的转化率非常高——搜这个词的人就是要用这个工具。

## 【PMF 验证方式】

判断 Termify 是否真的被需要，我看三个指标：

**第一，自发分享率。** 用户转换完成后，是否会下载并分享到社交平台？在页面上加一个"分享到 Twitter/微博"的按钮，统计点击率。如果超过 10% 的用户主动分享，说明产品有传播性。

**第二，复访率。** 用户是否会回来做第二次、第三次转换？通过生成唯一分享链接来追踪（不依赖登录系统）。如果同一个链接被多次访问，说明用户在反复使用。

**第三，社区反馈密度。** GitHub Issues 里是否出现"能不能支持 XX 格式""能不能加 XX 字符集"这类功能请求。有人提需求，说明他们真的在用，而且想要更多。

## 【定价模型】

Termify 的核心定位是开源玩具，商业化不是第一优先级，但未来有几条可行路径：

**永久免费层：** 基础功能完全免费开源——GIF/图片上传、5 种字符风格、Python 和 HTML 输出。这是社区信任的基础，也是增长的引擎。

**付费增值层（Pro）：** 按月订阅，面向重度用户和嵌入式开发者。包括：视频输入支持（MP4 转终端动画）、批量转换、高分辨率输出、自定义字符集上传、优先渲染队列。定价 $3-5/月，对标"一杯奶茶钱"的心理账户。

**硬件联名层：** 和开发板厂商（Arduino、Seeed Studio、DFRobot）合作，在官方套件里预置 Termify 生成的示例代码，品牌露出换资源。这不是直接收入，但能带来精准用户和品牌背书。

## 【Token 成本控制】

Termify 的一个核心优势是：当前 MVP 阶段几乎不消耗 LLM Token。图像到字符的转换基于 Floyd-Steinberg 抖动算法和像素映射，这是纯计算任务，不需要调用大模型。服务器成本主要是 CPU 和带宽，不涉及 AI 推理成本。

未来引入"AI 主体识别"功能（自动识别图片中的脸/主体，在字符映射时保留更多细节）时，才需要调用视觉模型。控制策略：

**分层调用。** 先用轻量级本地模型（如 MediaPipe）做主体检测，只有检测失败时才 fallback 到云端视觉 API。

**结果缓存。** 同一张图片的检测结果缓存 7 天，避免重复调用。

**免费额度限制。** 免费用户每天 10 次 AI 增强转换，Pro 用户不限量。既控制了成本，又制造了付费转化点。

**客户端计算优先。** 对于简单的亮度分析，直接在浏览器端用 Canvas API 完成，完全不经过服务器。

## 【下一步计划】

最想验证的一件事：**用户是否真的会把自己做的终端动画分享出去。**

如果分享行为成立，Termify 的增长飞轮就转起来了——用户做动画 → 分享 → 别人看到 → 也想做 → 新用户进来。下一步我会优先做两件事：一是在下载页加一个"生成预览 GIF + 分享链接"的功能，降低分享门槛；二是做一个 Gallery 页面，展示社区里最好的作品，让后来者有参考、有动力。

先让飞轮转起来，再考虑商业化。
